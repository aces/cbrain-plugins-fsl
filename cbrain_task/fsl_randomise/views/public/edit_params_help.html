How to run randomise:

Randomise allows you to run statistical tests for a given GLM, on many 4D input files. On the file menu, choose the following:
  • One or more 4D files
  • One mask file
  • A design matrix collection. Please note that the design matrix collection can be one directory, or it can be different files you get from a typical GLM design (i.e. files with extensions .mat, .con, .grp etc). Note that you can choose either a directory or several files. If a directory is chosen, then CBRAIN will automatically assign the files to the appropriate randomise option. Make sure that if you choose a design directory, then you have included all necessary design files.
CBRAIN options:
  • Number of permtation (-n): number of permutations (default 5000, set to 0 for exhaustive).
  • Cluster-based thresholding (-c): carry out cluster-based thresholding.
  • Threshold-Free Cluster Enhancement (-T): carry out Threshold-Free Cluster Enhancement.
  • f cluster thresholding (-F): carry out f cluster thresholding
  • Output voxelwise (-x): output voxelwise (corrected and uncorrected) p-value images.
  • Output raw (-R): output raw ( unpermuted ) statistic images.
  • Design matrix file (-d): design matrix file.
  • t contrasts file (-t): t contrasts file.
  • f contrasts file (-f): f contrasts file.
  • Exchangeability block labels file (-e): exchangeability block labels file.
  • Mask image (-m): mask image.
  • Input (-i): 4D input image.
  • Output (-o): output file-rootname.
